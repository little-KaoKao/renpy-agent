# Ren'Py 工具箱全量清单

**用途**：设计每个镜头前先扫一眼这份清单，确认没漏掉可用工具。
`effects-catalog.md` 按**情绪/场景**查"用什么"；这份按**类别**列**全部**，是工具箱的盘点。

难度：★=一行内置 / ★★=需素材或几行 ATL / ★★★=Shader、Live2D、3D 或自定义 Python。

## A. 视觉效果全清单

### 玩家交互字段书写规范（故事板里）

- **单选分支**（最常见）：

  ```
  玩家交互：
    类型：menu
    选项：
      - "告白" → label route_confession
      - "说点别的" → label route_friend
  ```

- **条件选项**：

  ```
  玩家交互：
    类型：menu
    选项：
      - "牵她的手" → label holding_hands（需要 affection >= 50）
      - "保持距离" → label keep_distance
  ```

- **限时选择**：

  ```
  玩家交互：
    类型：menu (限时 5 秒)
    超时行为：跳转到 label scene_silence
    选项：
      - "抓住她" → label save_her
      - "大喊警告" → label warn_her
  ```

- **文本输入**：

  ```
  玩家交互：
    类型：input
    提示："你叫什么名字？"
    变量名：player_name
    默认值："凛"
  ```

- **图片热区**：

  ```
  玩家交互：
    类型：imagemap
    底图：bg_room_detail
    热区：
      - 书桌 → label inspect_desk
      - 窗户 → label inspect_window
  ```

- **自定义 screen**：

  ```
  玩家交互：
    类型：custom_screen
    screen 名：phone_ui
    说明：主角查看手机，显示 3 条短信可点击
  ```

- **静默/定时**：

  ```
  玩家交互：
    类型：none (pause 3秒)
  ```

## C. 如何使用这份清单

1. **新镜头设计**：扫一眼 A1–A12 分类，确认**当前镜头可用的工具**，再从 `effects-catalog.md` 挑情绪契合的 1–2 个。
2. **交互点设计**：扫一眼 B 节，确认选择的交互类型是否存在更合适的变体（例如本来想用普通 `menu`，但其实场景需要"条件选项"或"限时"）。
3. **难度预警**：★★★ 工具在故事板里使用时，**必须**在"实施备注"里标注依赖（"需 Live2D 模型"、"需编写 GLSL shader"、"需 3D 分层素材"）。
4. **别叠加同类**：这份清单是工具箱不是菜单。一个镜头挑 1–2 个主要效果，加 0–1 个次要，避免"樱花 + 雪花 + 光斑 + 模糊 + 震动"的灾难。
5. **工具与情绪脱钩时停手**：如果你在清单里找一个"好看的"效果硬塞进镜头，而说不出"为什么这里用它"，那就是不该用。
