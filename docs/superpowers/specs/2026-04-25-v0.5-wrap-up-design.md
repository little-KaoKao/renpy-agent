## 2. Part A — 真 key smoke(`scripts/runninghub-smoke.mjs`)

### 2.1 命令形态

```
node scripts/runninghub-smoke.mjs [APPKEY...]
```

- **无参**:跑全部 8 个 AppKey
- **有参**:只跑列出的 AppKey,**自动补齐依赖**(下文 §2.3)
- **退出码**:全失败 = 1,至少一条成功 = 0

### 2.2 8 条 smoke case

每条 case 的定位:**用最小合法参数,让 RunningHub 真跑一次,产物下载到本地,留 json log**。

| AppKey | 模型 | role 最小参数 | 依赖 |
| --- | --- | --- | --- |
| `CHARACTER_MAIN_IMAGE` | MJ v7 文生图 | prompt = "a cat girl in school uniform" | - |
| `SCENE_BACKGROUND` | Nanobanana2 文生图 | prompt = "anime classroom at sunset" | - |
| `CHARACTER_EXPRESSION` | Nanobanana2 图生图 | prompt = "smiling face", reference_image_1 = SCENE_BACKGROUND 产物 URL | SCENE_BACKGROUND |
| `CHARACTER_DYNAMIC_SPRITE` | Seedance2.0 图生视频 | prompt = "gentle breathing", first_frame = CHARACTER_MAIN_IMAGE 产物 URL | CHARACTER_MAIN_IMAGE |
| `CUTSCENE_IMAGE_TO_VIDEO` | Seedance2.0 图生视频 | prompt = "slow camera pan", first_frame = SCENE_BACKGROUND 产物 URL | SCENE_BACKGROUND |
| `VOICE_LINE` | Qwen3 TTS | voice_text = "gentle female voice", line_text = "你好,初次见面" | - |
| `SFX` | Qwen3 TTS(复用) | voice_text = "ambient sound field, no voice, pure environmental audio", line_text = "rain on window" | - |
| `BGM_TRACK` | SunoV5 | title = "smoke-test", prompt = "piano ballad, melancholic" | - |

**注**:
- `CHARACTER_EXPRESSION` 的参考图 role 名在 schema 里是 `reference_image_1`(不是 `first_frame`)
- Schema 带 `defaultValue` 的 field(如 Seedance 的 `real_person_mode` / `duration` / `ratio` / `resolution`,SunoV5 的 `version`)不需要 caller 传
- `CHARACTER_EXPRESSION`(Nanobanana2 图生图)的 schema 里 `reference_image_1` **是 optional**(文生图模式允许缺图),但 smoke 强制传入,以验证"图生图"真正跑通

依赖型 AppKey(Expression / DynamicSprite / Cutscene)把**源 AppKey 的 remoteUri(不是本地路径)**喂给 `first_frame` role。

### 2.3 依赖 DAG 处理

依赖边共 3 条:

```
SCENE_BACKGROUND → CHARACTER_EXPRESSION
SCENE_BACKGROUND → CUTSCENE_IMAGE_TO_VIDEO
CHARACTER_MAIN_IMAGE → CHARACTER_DYNAMIC_SPRITE
```

**补齐算法**:用户传 `CUTSCENE_IMAGE_TO_VIDEO` 但没传 `SCENE_BACKGROUND` → 自动把 `SCENE_BACKGROUND` 加到运行集。拓扑排序后按序执行。

**源 AppKey 失败时**:依赖它的下游 AppKey 标记为 `status: "skipped"`,不报错,继续后续独立 case。

### 2.4 实现

**复用生产代码**,不再手写 nodeInfoList:

```js
import { HttpRunningHubClient } from '../dist/executers/common/runninghub-client.js';
import { RUNNINGHUB_APP_SCHEMAS } from '../dist/executers/common/runninghub-schemas.js';
import { runImageTask } from '../dist/executers/common/run-image-task.js';
```

(脚本先 `pnpm build` 再跑。README 里交代。)

每条 case 调一次 `runImageTask({ client, appKey, inputs: [{role, value}, ...] })`(签名见 [src/executers/common/runninghub-client.ts](../../../src/executers/common/runninghub-client.ts) 的 `RunningHubSubmitParams` 和 [run-image-task.ts](../../../src/executers/common/run-image-task.ts) 的 `RunImageTaskParams`),和 executer 用的同一条 helper。

`runImageTask` 成功返回 `{taskId, outputUri}`;smoke 脚本再调 `downloadAsset({remoteUrl: outputUri, gameDir: <smoke-dir>, targetRelativePath: '<APPKEY>.<ext>'})` 把产物落到本地。扩展名从 outputUri 推,推不出时用 fallback(图 `.png` / 视频 `.mp4` / 音频 `.mp3`)。

### 2.5 产物布局

```
runtime/smoke/<ISO8601-timestamp>/
├── summary.json                          # 汇总:8 行 {appKey, status, durationMs, taskId?, outputUri?, localFile?, error?}
├── CHARACTER_MAIN_IMAGE.json             # 单条 case 的完整结果
├── CHARACTER_MAIN_IMAGE.<ext>            # 下载的真产物(图/视频/音频)
├── SCENE_BACKGROUND.json
├── SCENE_BACKGROUND.<ext>
└── ...
```

每条 case json 结构:

```json
{
  "appKey": "CHARACTER_MAIN_IMAGE",
  "status": "ok" | "error" | "skipped",
  "taskId": "rh_xxx",
  "durationMs": 45123,
  "outputUri": "https://rh-outputs.../xxx.png",
  "localFile": "CHARACTER_MAIN_IMAGE.png",
  "error": null
}
```

### 2.6 错误处理

- RunningHub 4xx/5xx / poll 超时 / outputs 空 / 下载失败 → status = "error",error 写消息,**继续下一条**
- 依赖源失败 → 下游 status = "skipped",不计入 error
- 整脚本退出:有任一 status === "ok" → 0,否则 1

### 2.7 测试

Smoke 脚本本身**不进 `pnpm test`**。它就是端到端真调用,靠手跑验证。

## 3. Part B — 修改闭环 CLI + rebuild

### 3.1 命令形态

子命令分发:

```
renpy-agent generate <inspiration> [--name <slug>] [--audio-ui]      # 当前默认行为
renpy-agent modify character <story> --name <name> --description "..." [--rebuild]
renpy-agent modify dialogue  <story> --shot <N> --line <i> --text "..." [--rebuild]
renpy-agent modify shots     <story> --order 3,1,2,4,5,6,7,8 [--rebuild]
renpy-agent rebuild <story>
renpy-agent -h | --help
```

**兼容性**:argv[0] 不匹配任何子命令时,视为旧 `renpy-agent <inspiration>` 形式,走 generate handler。README 里交代。

### 3.2 parseArgs 重构

```ts
export type ParsedCliCommand =
  | { kind: 'generate'; inspiration: string; storyName?: string; audioUi?: boolean }
  | { kind: 'modify'; op: 'character'; storyName: string; characterName: string; description: string; rebuild: boolean }
  | { kind: 'modify'; op: 'dialogue'; storyName: string; shotNumber: number; lineIndex: number; text: string; rebuild: boolean }
  | { kind: 'modify'; op: 'shots'; storyName: string; order: readonly number[]; rebuild: boolean }
  | { kind: 'rebuild'; storyName: string }
  | { kind: 'help' };

export function parseArgs(argv: readonly string[]): ParsedCliCommand;
```

**parseArgs 内部结构**:顶层 dispatcher 按 argv[0] 分派到 `parseGenerateArgs` / `parseModifyArgs` / `parseRebuildArgs`。每个子 parser 自己处理自己的 flag,非法 flag 抛 Error。

`parseModifyArgs` 按 argv[1](`character` / `dialogue` / `shots`)再分派到三个 op 的 parser,每个 op 独立校验自己的 required flag。

### 3.3 新增:`src/pipeline/rebuild.ts`

```ts
export interface RebuildOptions {
  readonly storyName: string;
  readonly runtimeRoot?: string;     // 默认 process.cwd() + /runtime
  readonly sdkRoot?: string;         // 传给 runQa
}

export interface RebuildResult {
  readonly gamePath: string;
  readonly testRun: TestRun;
}

export async function rebuildGameProject(opts: RebuildOptions): Promise<RebuildResult>;
```

流程:

1. 解析 `gameDir = <runtimeRoot>/games/<storyName>/game`(和 [run-pipeline.ts](../../../src/pipeline/run-pipeline.ts) 用的同一个路径约定)
2. `loadStoryWorkspace(gameDir)` → 拿 `StoryWorkspaceSnapshot` { planner, writer, storyboarder }(现有 API,[src/pipeline/workspace.ts:128](../../../src/pipeline/workspace.ts#L128))
3. 读 `<runtimeRoot>/games/<storyName>/asset-registry.json`(可能不存在 → 传 undefined);注意 registry 在 `game/` 的 **兄弟位**,不在 `game/` 内(PLAN 记载:避开 Ren'Py 的 `.rpa` 打包扫描)
4. `generateGameProject(planner, storyboarder, registry?)` → `.rpy` 文件集合
5. `writeGameProject(gameDir, rpyFiles)` → 落文件 + 拷 templates
6. `runQa({gameDir, sdkRoot})` → lint

参数 `runtimeRoot` 缺省值与 CLI 保持一致(`process.cwd() + '/runtime'`)。

**不跑** Planner/Writer/Storyboarder,**不跑** AudioUiStage。

### 3.4 扩展 `src/pipeline/modify.ts`:`applyModification` wrapper

三个现有函数签名**不改**(测试已覆盖)。新增一个统一入口给 CLI 用:

```ts
export type ModifyOperation =
  | { op: 'character'; characterName: string; description: string }
  | { op: 'dialogue'; shotNumber: number; lineIndex: number; text: string }
  | { op: 'shots'; order: readonly number[] };

export async function applyModification(opts: {
  storyName: string;
  operation: ModifyOperation;
  runtimeRoot?: string;
}): Promise<void>;
```

内部:`loadStoryWorkspace` → 按 op 分派到对应 modify 函数 → `saveStoryWorkspace`。AssetRegistry 回落由 `modifyCharacterAppearance` 内部自行处理(现有逻辑已经做了)。

### 3.5 cli.ts 主流程

```ts
async function main(argv) {
  let cmd: ParsedCliCommand;
  try { cmd = parseArgs(argv); } catch (err) { /* 打印错误 + usage,return 1 */ }

  switch (cmd.kind) {
    case 'help':     return printHelp();
    case 'generate': return runGenerate(cmd);
    case 'modify':   {
      await applyModification({ storyName: cmd.storyName, operation: toOperation(cmd) });
      console.log(`✅ Modified ${cmd.op} in ${cmd.storyName}`);
      if (cmd.rebuild) {
        const r = await rebuildGameProject({ storyName: cmd.storyName });
        console.log(`✅ Rebuilt. Game at: ${r.gamePath}`);
        return r.testRun.result === 'fail' ? 2 : 0;
      }
      console.log(`   Run \`renpy-agent rebuild ${cmd.storyName}\` to regenerate script.rpy`);
      return 0;
    }
    case 'rebuild': {
      const r = await rebuildGameProject({ storyName: cmd.storyName });
      console.log(`✅ Rebuilt. Game at: ${r.gamePath}`);
      return r.testRun.result === 'fail' ? 2 : 0;
    }
  }
}
```

### 3.6 错误处理

- 子命令缺参(story 名、--name、--shot、--order 等)→ parseArgs 抛 Error → main 捕获 → 退出 1 + 打印对应子命令 usage
- `loadStoryWorkspace` 目录不存在 → 现有 API 抛 Error → 改成 CLI 友好消息:`Story '<name>' not found. Did you run \`renpy-agent generate\` first?` 退出 1
- `modifyCharacterAppearance` 找不到角色 / `modifyDialogueLine` 越界 / `reorderShots` 非全排列 → 现有函数抛 Error → CLI 捕获 → 打印错误信息 + 退出 1
- `--rebuild` 时 QA result === 'fail' → 退出 2(和 generate 一致)
- `rebuild` 时 QA result === 'fail' → 退出 2
- `modify --rebuild` 里 modify 成功但 rebuild 阶段抛异常:**snapshot 已写盘,不回滚**,打印 "Modified but rebuild failed: ..." + 提示可以手动 `renpy-agent rebuild <story>` 重试。退出 1。
