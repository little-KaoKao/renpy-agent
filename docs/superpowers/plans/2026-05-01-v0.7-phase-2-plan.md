## 4. 是否用 worktree?

| 分支 | 用 worktree? | 为什么 |
|---|---|---|
| **v6-smoke** | **否** | 只写报告,不改代码;在 main 开个临时 branch 跑完就合 |
| **budget-cap** | **否** | 改动小(~100 行),在 main 直接切分支即可;干扰 main 的风险低 |
| **storyboarder-enum** | **是** | 改 coder.ts 热点,想继续用 main 跑别的事时会互相踩,隔离省事 |
| **task-agent-real** | **是** | 3-5 天工作,新建 3 个文件 + 改测试,开发中会跑很多次 smoke;隔离能避免污染 main 的 dist/ 和 runtime/ |
| **modify-chain** | **是** | 3-5 天 + 依赖 task-agent-real 产物;独立 worktree 方便在其中 `pnpm build && pnpm start` 反复试 |

**Worktree 创建命令**(子会话开头跑):

```powershell
# PowerShell (Windows)
$branch = "refactor/storyboarder-enum"   # 换成你的分支名
$slug = $branch -replace '^(refactor|feat|fix|chore)/', ''
cd E:\RenPy
git fetch origin
git worktree add ".worktrees\$slug" -b $branch origin/main
cd ".worktrees\$slug"
pnpm install   # node_modules 不共享,必须重装
```

```bash
# bash (macOS/Linux 参考)
branch="refactor/storyboarder-enum"
slug=${branch#*/}
cd ~/RenPy
git fetch origin
git worktree add ".worktrees/$slug" -b "$branch" origin/main
cd ".worktrees/$slug"
pnpm install
```

**Worktree 清理**(合并后):
```powershell
cd E:\RenPy
git worktree remove ".worktrees\<slug>"
git branch -d <branch>           # 本地
git push origin --delete <branch> # 远程
```
