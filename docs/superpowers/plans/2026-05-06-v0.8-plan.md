## 1. 根因分析

### 1.1 character 主图未触发（Minor-3）

根因：`run-v5.ts` 的 `buildDefaultTaskAgents` 在 `runningHubClient` 未注入或 `RUNNINGHUB_API_KEY` 未设时强制 `DRY_RUN`。M8 smoke 脚本 `v5-real-key-smoke.mjs` 没有传 `runningHubClient`。

代码路径：
```
run-v5.ts → buildDefaultTaskAgents(dryRun=true) → forceDryRun(characterMainImageGenerator)
→ call_task_agent → status: 'dry_run' → Planner 看到 dry_run，继续
```

修复：smoke 脚本注入 `runningHubClient`（已有 `HttpRunningHubClient`，只是没传进去）。

### 1.2 Tier 2 POC 从未被 handoff

根因：`PLANNER_RULES` 的 Stage A order 只列了 Tier 1 POC，没有提 Tier 2。Planner 在 qa finish 后直接 `output_with_finish("Stage A delivered")`，不知道还有 Tier 2 工作。

修复：在 Planner system prompt 里加 Stage B 描述，告诉 Planner：Stage A 完成后，如果有 `RUNNINGHUB_API_KEY`，继续 handoff Tier 2 POC。

- `src/agents/run-v5.ts`（确认 `canRunReal` 逻辑正确）

- `src/executers/task-agents/main-image-generator.ts`（已实现，不动）

**具体做法：**

1. `v5-real-key-smoke.mjs` 里构造 `HttpRunningHubClient` 并传给 `runV5()`：
   ```js
   import { HttpRunningHubClient } from '../dist/executers/common/runninghub-client.js';
   const runningHubClient = new HttpRunningHubClient(process.env.RUNNINGHUB_API_KEY);
   await runV5({ ..., runningHubClient });
   ```

2. 确认 `run-v5.ts` 的 `canRunReal` 判断：`params.runningHubClient !== undefined && hasRhKey`。两个条件都满足时用真实 task agents。

**验收：**
- [ ] `pnpm build` 干净

### 3.2 feat/planner-tier2-handoff

**允许碰：**
- `src/agents/planner.ts`（`PLANNER_RULES` 加 Stage B 段）
- `src/agents/poc-registry.ts`（Tier 2 description 补充细节）

**绝对不碰：**
- 任何 Tier 2 `tools.ts`（已实现，不动）
- `src/agents/executer.ts`
- `src/llm/claude-client.ts`

**具体做法：**

在 `PLANNER_RULES` 的 Rule 5（finish 条件）之后加 Stage B 段：

```
Stage B (optional, runs after Stage A is complete):
After qa has passed lint, if RUNNINGHUB_API_KEY is available (you will see
"tier2_available: true" in the workspace index), continue with Tier 2 POCs:
  - music_director: generate BGM tracks for each scene mood
  - voice_director: generate voice lines for key dialogue shots
  - sfx_designer: generate ambient SFX for scene transitions
  - ui_designer: generate UI patches (title screen, textbox style)
Stage B order: music_director → sfx_designer → voice_director → ui_designer → coder → qa
After Stage B qa passes, call output_with_finish with taskSummary="Stage A+B delivered".
If RUNNINGHUB_API_KEY is not available, skip Stage B and finish with "Stage A delivered".
```

`workspace-index.ts` 加一行 `tier2_available: boolean`（读 `process.env.RUNNINGHUB_API_KEY`），让 Planner 能感知。

**验收：**
- [ ] `pnpm test` 全绿
